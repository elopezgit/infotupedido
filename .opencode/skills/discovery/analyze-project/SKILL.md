# analyze-project

## Objetivo
Analizar un proyecto completo para entender su estructura, tecnologías, calidad y estado general. Es la skill de entrada para cualquier proyecto existente.

## Cuándo usar
- Primera vez que se trabaja en un proyecto
- Después de un período largo sin actividad
- Cuando se necesita una visión general del estado del proyecto

## Entradas
- Código fuente del proyecto
- Archivos de configuración

## Procedimiento
1. Analizar estructura de carpetas y organización
2. Detectar lenguajes, frameworks y librerías
3. Identificar tipo de BD, ORM, migraciones
4. Revisar configuración de CI/CD
5. Evaluar cobertura de pruebas y frameworks de testing
6. Identificar dependencias desactualizadas o vulnerables
7. Detectar secretos hardcodeados o .env comiteados
8. Revisar calidad de código (lint, formateo, complejidad)
9. Estimar deuda técnica general

## Salida
- Resumen ejecutivo del proyecto
- Stack tecnológico detectado
- Estado de calidad (testing, deuda técnica, dependencias)
- Riesgos detectados (seguridad, secretos, vulnerabilidades)
- Recomendaciones iniciales
