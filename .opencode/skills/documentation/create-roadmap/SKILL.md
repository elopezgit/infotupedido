# create-roadmap

## Objetivo
Crear y mantener un roadmap del proyecto con hitos, dependencias, prioridades y estimaciones.

## Cuándo usar
- Proyecto nuevo (roadmap inicial)
- Fin de sprint o hito (actualizar roadmap)
- Cuando cambian las prioridades
- Como parte de la documentación del proyecto

## Entradas
- PROJECT_PROFILE (visión y objetivos)
- PROJECT_MEMORY (historial de sesiones)
- Conversación con el usuario sobre prioridades

## Procedimiento
1. Listar tareas pendientes y features planificadas
2. Agrupar por hitos o versiones
3. Identificar dependencias entre tareas
4. Estimar esfuerzo relativo (bajo/medio/alto)
5. Priorizar por valor de negocio vs esfuerzo
6. Asignar a fases del proyecto
7. Actualizar ROADMAP.md y PROJECT_MEMORY

## Salida
- Roadmap actualizado en memory/ROADMAP.md
- Hitos con fechas estimadas y dependencias
- Tareas priorizadas por valor/esfuerzo
- Próximos pasos recomendados
